## Project Overview

HumaneBench website for Building Humane Tech – a Vue 3 SPA presenting benchmark results that measure whether AI models support human flourishing. Displays interactive score visualizations, research whitepapers, case studies, and humane AI principles.

## Commands

```bash
npm install          # Install dependencies
npm run serve        # Dev server at http://localhost:8080 (hot reload)
npm run build        # Production build to dist/
npm run lint         # ESLint with auto-fix
```

No test runner is configured.

## Tech Stack

- **Framework:** Vue 3 + TypeScript (strict mode)
- **UI:** Vuetify 3 (Material Design components) + Tailwind CSS 3 (utilities only, preflight disabled)
- **Routing:** Vue Router 4 with `createWebHistory`
- **Icons:** Material Design Icons (`@mdi/font`, `@mdi/js`)
- **Content:** Markdown files parsed by `marked`, loaded via `raw-loader`
- **Build:** Vue CLI (Webpack), Babel
- **Package manager:** npm

## Architecture

### Routing & Layout

All routes are children of `DefaultLayout` (`src/layout/default.vue`), which provides the persistent header (app bar + nav) and footer. Routes are defined in `src/router/index.ts`.

Current routes:
- `/` – Home (MarkdownPage, pageName: 'index')
- `/whitepaper` – Whitepaper (MarkdownPage, pageName: 'whitepaper')
- `/principles` – Principles (MarkdownPage, pageName: 'principles')
- `/case-studies` – Case Studies (CaseStudiesPage)
- `/models` – Models (ModelsPage)

**To add a new page:** Create a view component in `src/views/`, add a route in `src/router/index.ts` as a child of the DefaultLayout route, and add a nav link in `src/layout/default.vue` (both the desktop nav buttons and the mobile navigation drawer).

### Content Pattern: Markdown with Embedded Components

`MarkdownPage.vue` (`src/views/`) is a reusable view that loads markdown from `src/pages/*.md`, parses it to HTML, then mounts Vue components into `<div data-component="ComponentName">` placeholders. Supported components: ScoreGrid, ScoreCarousel, Events, News, InputOutputExamples, WhitepaperButton.

### Benchmark Visualizations

SVG figures and JSON data in `public/figures/` are built separately in the [humanebench repo](https://github.com/buildinghumanetech/humanebench) and copied here. `ScoreGrid.vue` loads these SVGs and adds interactive hover tooltips.

### Model Score Data

`public/data/model_scores.json` contains extracted score data for 15 models across 3 datasets (baseline, good_persona, bad_persona), grouped by provider. Generated by `scripts/extract-scores.js` which parses the SVG score grids. Re-run the script when SVGs are updated from the humanebench repo.

### Path Alias

`@/` maps to `src/` in both TypeScript and webpack (e.g., `import Foo from '@/components/Foo.vue'`).

## Styling Conventions

- **Background color:** `#fff4e0` (warm cream) used across header, footer, and page backgrounds
- **Link/accent color:** `#5539EC` (purple)
- **Typography:** Lora (serif, Google Fonts) for headings; Vuetify defaults (Roboto) for body
- **Layout:** Use Vuetify grid (`v-row`, `v-col`) for page structure, Tailwind utilities for spacing/helpers
- **Responsive:** Vuetify breakpoints (xs/sm/md/lg/xl) plus custom media queries at 768px and 480px
